### 1.7 利用料金（最重要）

#### 本学構成員（愛媛大学の役員・教職員・学生）

**原則として無償です。**

活動に必要な実費（教材費・消耗品費等）については、利用者が参加者から徴収することを認めます。

#### 学外利用者（企業・NPO・個人等が主催者となる場合）

**施設使用料：1 時間あたり 1,160 円**

#### 展示利用の特例【最重要】

**展示会期中は利用料金がかかりません（無料）。**

ただし、以下の準備・撤去費用が必須となります：

- **展示準備：** 開催日前日の 1 日分（午前 10:00 〜 午後 16:30 = 6.5 時間）
- **展示撤去：** 展示終了日翌日の 1 日分（午前 10:00 〜 午後 16:30 = 6.5 時間）
- **合計：** 13 時間分 = **15,080 円**（1,160 円 × 13 時間）

※実際の展示期間の長さは関係なく、準備・撤去分の料金は必ずお支払いいただきます。

#### 展示貸出期間

展示として貸し出す最長期間は **21 日間** です。  
準備・撤去の作業日はミュージアム休館日を除きます。

#### 使用料の返還・減免

- 既納の使用料は、原則として返還しません。  
  （ただし、大学の都合や不可抗力により施設が利用できなくなった場合は除く）
- Imagine Deck 運営会議が特例として認めた場合に限り、使用料の全部または一部を免除することができます。

**施設使用料の徴収・管理事務は、ミュージアム事務（広報課）が担当します。**

## 2. 料金・準備/撤去・競合判定の仕様

### 2.1 展示利用における準備・撤去日の自動計算・競合判定

#### 業務上の要件（事務 広報課からの質問に対する回答）

**質問：**  
利用者が展示期間のみを入力すれば、システムが自動的に準備日・撤去日を判定し、その日に既に他の使用者がいる場合はエラー表示できるようにしたい。

**回答：**  
**実装可能です。** 以下の仕様に従ってください。

---

#### 2.1.1 システムの振る舞い

**ユーザー入力：**
- 展示開始日（YYYY-MM-DD）
- 展示終了日（YYYY-MM-DD）

**システムが自動計算：**
1. **準備日：** 展示開始日の前日（開室日）  
   - 最新の開室日（日曜でない、祝日でない、ミュージアム休館日でない）を遡って検索
2. **撤去日：** 展示終了日の翌日（開室日）  
   - 最初の開室日（日曜でない、祝日でない、ミュージアム休館日でない）を順方向に検索

**競合チェック：**
- 準備日と撤去日について、既存予約（events.json から Firestore へ移行後）を確認
- 該当日に ANY の reservation/event/maintenance が存在する場合 → エラー表示
- 時間枠ではなく「その日の全体」を貸切（exclusive）として判定

**費用計算：**
- 準備日 1 日（6.5 時間） + 撤去日 1 日（6.5 時間） = 13 時間
- 費用：13 時間 × 1,160 円/時間 = **15,080 円（固定）**
- 展示期間中（開始日～終了日）は無料

---

#### 2.1.2 実装上の詳細

**開室日判定関数：**

```javascript
function isOpenDay(dateObj) {
  // dateObj は JS Date
  const dateStr = formatDate(dateObj); // YYYY-MM-DD
  const isSunday = dateObj.getDay() === 0;
  const isHoliday = isClosed(dateStr); // 既存関数を利用
  return !(isSunday || isHoliday);
}
```

**準備日計算：**

```javascript
function findPrepDay(exhibitionStartStr) {
  // exhibitionStartStr = "2026-06-15"
  const [y, m, d] = exhibitionStartStr.split('-').map(Number);
  const startDate = new Date(y, m - 1, d);
  
  let prepDate = new Date(startDate);
  prepDate.setDate(prepDate.getDate() - 1);
  
  // 遡って最初の開室日を見つける
  while (prepDate.getTime() > 0) {
    if (isOpenDay(prepDate)) {
      return formatDate(prepDate);
    }
    prepDate.setDate(prepDate.getDate() - 1);
  }
  return null; // エラー：準備日が見つからない
}
```

**撤去日計算：**

```javascript
function findTeardownDay(exhibitionEndStr) {
  // exhibitionEndStr = "2026-06-25"
  const [y, m, d] = exhibitionEndStr.split('-').map(Number);
  const endDate = new Date(y, m - 1, d);
  
  let teardownDate = new Date(endDate);
  teardownDate.setDate(teardownDate.getDate() + 1);
  
  // 順方向で最初の開室日を見つける（最大90日先まで）
  const maxSearch = new Date(endDate);
  maxSearch.setDate(maxSearch.getDate() + 90);
  
  while (teardownDate.getTime() <= maxSearch.getTime()) {
    if (isOpenDay(teardownDate)) {
      return formatDate(teardownDate);
    }
    teardownDate.setDate(teardownDate.getDate() + 1);
  }
  return null; // エラー：撤去日が見つからない
}
```

**競合チェック関数：**

```javascript
async function checkPrepTeardownConflict(prepDay, teardownDay) {
  // Firestore から予約一覧を取得
  const allReservations = await loadReservations(); // Firestore API
  
  const prepConflicts = allReservations.filter(r => r.date === prepDay);
  const teardownConflicts = allReservations.filter(r => r.date === teardownDay);
  
  if (prepConflicts.length > 0 || teardownConflicts.length > 0) {
    const conflicts = [];
    if (prepConflicts.length > 0) conflicts.push(`準備日 ${prepDay}`);
    if (teardownConflicts.length > 0) conflicts.push(`撤去日 ${teardownDay}`);
    
    return {
      hasConflict: true,
      message: `${conflicts.join(' と ')} は既に他の利用者に予約されています。`
    };
  }
  
  return { hasConflict: false };
}
```

### 5.3 Firebase Hosting での Cache 設定

`firebase.json` に以下を追加：

```json
{
  "hosting": {
    "headers": [
      {
        "source": "**/*.@(css|js|jpg|jpeg|png|gif|svg|woff|woff2)",
        "headers": [
          { "key": "Cache-Control", "value": "public, max-age=31536000" }
        ]
      },
      {
        "source": "*.html",
        "headers": [
          { "key": "Cache-Control", "value": "public, max-age=3600" }
        ]
      }
    ]
  }
}
```

## 7. 要確認・要依頼事項（オーナー）

### 7.1 Naming の揺れ確認

**現状：** 公式ガイドラインでは「交流スペース」が基本だが、施設概要欄に「体験スペース」の表記あり。

**要確認：** どちらを統一名称とするか、愛媛大学ミュージアム / 広報課に最終確認。

**現在の方針：** 「交流スペース」を採用（ガイドライン vol6 に準拠）

---

### 7.2 Favicon 素材

**要依頼：** 
- ロゴの「i」部分（渦巻きオーブ）をmark として独立させた
- 192×192px、32×32px の PNG / SVG ファイルの生成
- デザイナーに Figma / AI で出力を依頼

### 7.3 展示関連の細則

**未定事項（ガイドラインでは「別途定める」）：**

**対応：** 運営会議で細則を別途策定し、site に掲載する場合は UPDATE_NEEDED flag として mark

### 7.4 Firebase Emulator vs. 本番環境のデプロイタイミング

**Phase 1 現状：** Firebase Emulator でローカル開発  
**Phase 2 へのゲート：**
- Firestore schema の最終確認
- 広報課による審査・承認フロー整備
- 仮承認メール送信ロジック（Firebase Functions）の実装・テスト
- Google Search Console への登録・サイトマップ送信

**デプロイ予定：** 2026 年 6 月下旬以降（所有者ゲート）

### 7.5 横展開（他大学サービス連携）の可能性

**現状：** Imagine Deck は愛媛大学固有の施設。

**将来展開の際に必要な対応：**
- 複数大学への汎用化検討時は、施設名称・カラー・ロゴを変数化する仕様に改造
- Firebase Firestore のマルチテナント化（大学 ID 単位での分離）
- 認証・課金体系の汎用化

**現在の優先度：** 低（Phase 3 以上の課題）
